# 角色卡使用说明（简化版）

## 一、用途

本角色卡用于 AI 跑团 / 互动叙事。
角色信息分为两类：

- **规则信息**：用于判定、战斗、成功与失败
- **叙事信息**：用于推动剧情、塑造人物行为

JSON 只保存**必要数据**，所有使用规则以本文档为准。

---

## 二、角色卡 JSON 模板

```json
{
  "id": "",
  "name": "",
  "attributes": {
    "str": 0,
    "dex": 0,
    "int": 0
  },
  "hp": 0,

  "personality": "",
  "motivation": "",
  "strengths": [],
  "weaknesses": [],
  "commentary": ""
}
```

---

## 三、字段使用说明

### 1. id（唯一标识）
- 用于文件命名与读取
- 需稳定、唯一，建议只使用英文/数字/下划线

### 2. 属性（attributes）
- **str（力量）**：近战、承伤、体力
- **dex（敏捷）**：闪避、先手、远程
- **int（智力）**：判断、技能、魔法

属性只决定**成功概率**，不决定行为意愿。

---

### 3. hp（生命值）
- 表示角色还能承受多少伤害
- hp ≤ 0 时，角色失去行动能力（具体后果由剧情决定）

---

### 4. personality（性格）
- 决定角色在不确定情况下的**行为倾向**
- 行动选择应尽量符合性格描述

---

### 5. motivation（动机）
- 角色的长期目标或执念
- 剧情推进应优先围绕该动机展开

---

### 6. strengths（优点）
- 用于解释成功的合理性
- 在合适情境下可被主动调用

---

### 7. weaknesses（缺点）
- 用于制造失败、冲突和代价
- 在关键抉择或压力下应优先触发

---

### 8. commentary（评语）
- 用于整体人物理解和叙述基调
- 不参与任何判定

---

## 四、综合使用原则（必须遵守）

在生成剧情或行动时：

1. 行为应符合 **personality**
2. 剧情应推动或阻碍 **motivation**
3. 成功可关联 **strengths**
4. 失败与代价优先关联 **weaknesses**
5. 整体叙述参考 **commentary**

---

## 五、禁止事项

- 不得新增未定义字段
- 不得忽略缺点只描述成功
- 不得将叙事字段数值化
